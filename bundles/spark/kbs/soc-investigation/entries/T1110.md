# Brute Force

**What it is:** Adversary guesses credentials at scale — classic brute force, password spraying (one password across many accounts), or credential stuffing (breached creds replayed).

**How it manifests:**
- Auth logs: bursts of 4625 (failed logon) — many failures for one account (brute force) or one-to-few failures across MANY accounts from a single source (spray, evades lockout). Kerberos 4771 pre-auth failures (0x18). A failure storm followed by a single 4624 success is the breakthrough signal.
- Cloud/IdP: high-volume sign-in failures from one IP/ASN, often from VPN/Tor/cloud-hosting ranges; legacy-auth endpoints (IMAP/SMTP/EWS) targeted because they bypass MFA.
- Logon types: type-3 (network) and type-10 (RDP) failures dominate; SMB/RDP/OWA/VPN are common targets.

**Investigation steps (in order):**
1. Aggregate 4625/4771 by source IP and by target account in the window — distinguish spray (many accounts, one IP) vs. brute force (one account).
2. Find the breakthrough: any 4625-storm account that then shows a 4624 success from the same source IP = compromised; pivot to T1078.
3. Identify the targeted surface (RDP/VPN/OWA/legacy-auth) and whether MFA was enforced.
4. Check IdP for the source IP/ASN reputation and any successful sign-ins it produced.
5. Contain: block source IP, reset breached accounts, revoke sessions, disable legacy auth.
6. Scope all accounts touched by the same source to find every success.

**Data sources:** Windows Security 4625/4624/4771, IdP/SSO sign-in logs (Azure AD/Okta with failure reasons), VPN + RDP-gateway logs, firewall/perimeter logs, authentication logs for OWA/EWS/IMAP.

**Pivot/related:** T1078 (successful login from guessed creds), T1021.001 (RDP target), T1133 (external remote services), T1556 (MFA/auth-process tamper).
