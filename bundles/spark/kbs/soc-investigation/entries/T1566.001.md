# Spearphishing Attachment

**What it is:** Adversary emails a weaponized attachment (macro doc, ISO/IMG, OneNote, LNK, HTA, malicious PDF) that executes code when the user opens it.

**How it manifests:**
- Process tree: `OUTLOOK.EXE` (or a browser/`explorer.exe` from a saved attachment) spawns `WINWORD.EXE`/`EXCEL.EXE`/`onenote.exe`, which spawns `cmd.exe`, `powershell.exe`, `mshta.exe`, `wscript.exe`, `rundll32.exe`, or `regsvr32.exe`. Office → script-host is the highest-signal indicator.
- Command lines: encoded PowerShell (`-enc`, `-w hidden`), `mshta http(s)://`, `rundll32 ... ,DllRegisterServer`, curl/certutil downloading a payload.
- File artifacts in `%TEMP%`, `%APPDATA%`, `Downloads`, or Outlook's `Content.Outlook` cache; ISO/IMG mounts presenting a hidden LNK; OneNote-embedded scripts.
- Network: child process beacons to a freshly-registered or low-reputation domain shortly after document open.

**Investigation steps (in order):**
1. Identify patient-zero email — sender, subject, attachment hash/name; pull the original message from the mail security gateway.
2. Confirm the parent-child chain in EDR: did an Office/PDF/archive app spawn a script interpreter or LSASS-touching binary?
3. Detonate the attachment hash in a sandbox / check VT for C2 and dropped-file IOCs.
4. Pivot on the C2 domain/IP across proxy + DNS to scope every host that resolved/connected.
5. Search mail logs for the same sender/subject/hash to find all recipients (campaign scope).
6. Check for follow-on persistence (run keys, scheduled tasks) and credential access on patient zero.

**Data sources:** Email security gateway logs, EDR process-creation events, proxy/web logs, DNS resolution logs, file-creation telemetry, mail-server message tracking.
