# Server Software Component: Web Shell

**What it is:** Adversary plants a script (ASPX/JSP/PHP) in a web server's content directory to gain a persistent, web-accessible backdoor for command execution.

**How it manifests:**
- File artifacts: a new/modified `.aspx`/`.asp`/`.jsp`/`.php` in the webroot with a recent timestamp, small or obfuscated content (eval/`Request`/`cmd`/base64), filenames masquerading as legit (`error.aspx`, `logout.jsp`) or random.
- Process tree: the web service account (`w3wp.exe`, `tomcat`/`java`, `httpd`, `php-fpm`) spawning `cmd.exe`/`powershell.exe`/`bash`/`whoami`/`net.exe` — a web worker spawning a shell is the canonical web-shell signal.
- Web logs: POST requests (often) to the shell URL with anomalous parameters, from one or few external IPs, returning 200; access to a page that didn't exist before a given date.
- Often the follow-on to T1190 (exploit) — the exploit writes the shell, then the actor returns via it.

**Investigation steps (in order):**
1. From EDR, find the web service account spawning command interpreters; the child command lines reveal attacker actions.
2. Identify the script file the requests hit; pull it, hash it, and inspect for eval/command-exec patterns.
3. In web/IIS logs, enumerate every request to that URL — source IPs, methods, timeline — to bound activity and find the planting request.
4. Hunt the entire webroot for files created/modified after the suspected compromise date (sort by mtime) to find sibling shells.
5. Determine how it was written (correlate file-create time with the exploit request — T1190) and whether creds were harvested.
6. Scope other web servers for the same filename/hash/IP.

**Data sources:** EDR process-creation (web-service-account → shell), file-creation/modification in webroot, IIS/Apache/Nginx access logs, WAF logs, file-integrity-monitoring on the content directory.

**Pivot/related:** T1190 (exploit that drops the shell), T1059 (command execution via the shell), T1003 (credential dumping next), T1078 (harvested accounts), T1133 (external remote services).
