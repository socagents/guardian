# Application Layer Protocol: DNS

**What it is:** Adversary uses DNS queries/responses as a covert C2 channel (and sometimes exfil) — encoding data in subdomains and TXT/NULL/CNAME records to slip past egress controls that allow DNS.

**How it manifests:**
- DNS logs: high volume of queries to subdomains of a single registered domain; long, high-entropy/base32/hex-looking labels (`a8f3...e2.c2.attacker.com`); unusual record types (TXT, NULL, CNAME) at scale; many unique subdomains under one parent (NXDOMAIN bursts).
- Beacon-like periodicity — queries at regular intervals; consistent query length; a single internal host generating disproportionate DNS volume.
- Resolution to a domain whose authoritative NS is attacker-controlled and recently registered.
- Process telemetry: a non-browser process (or `nslookup`/`powershell`) issuing the queries.

**Investigation steps (in order):**
1. In DNS logs, rank internal hosts by query volume and by count of unique subdomains per parent domain — outliers are candidates.
2. Inspect the parent domain: registration age, NS records, reputation; examine label entropy/length and record types.
3. Identify the querying process on the host via EDR (correlate DNS event to PID) — a script/unknown binary is the tell.
4. Assess beacon periodicity and total bytes implied (subdomain length × query count) to gauge exfil.
5. Sinkhole/block the C2 domain and scope every host that queried it.
6. Pivot to the host's initial access + payload that established the channel.

**Data sources:** DNS resolver/query logs (full query name + record type), EDR DNS-request events (Sysmon Event 22) tied to process, proxy logs (to confirm DNS is the only egress used), passive-DNS/threat-intel on the domain.
