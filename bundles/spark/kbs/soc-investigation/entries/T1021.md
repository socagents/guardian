# Remote Services: RDP and SMB/Windows Admin Shares

**What it is:** Adversary moves laterally using valid credentials over Remote Desktop (T1021.001) or via SMB administrative shares like `C$`/`ADMIN$`/`IPC$` (T1021.002), often with PsExec-style remote execution.

**How it manifests (RDP):**
- Auth logs: 4624 **type 10** (RemoteInteractive) on the target; 4778/4779 (session reconnect/disconnect); TerminalServices-RemoteConnectionManager/LocalSessionManager 1149/21/22/25.
- A workstation initiating RDP outbound to many internal hosts (lateral fan-out) — unusual for a non-jump-box.

**How it manifests (SMB admin shares):**
- Auth logs: 4624 **type 3** (network) to `ADMIN$`/`C$`; 5140/5145 (network-share access) referencing admin shares; service install (7045 / Security 4697) of a randomly-named service = PsExec.
- File artifacts: payload copied to `C:\Windows\` via `ADMIN$`; PsExec's `PSEXESVC.exe`; remote `sc`/`wmic`/`Invoke-Command`/`at` execution.

**Investigation steps (in order):**
1. On the alerting host, pull 4624 and separate type-10 (RDP) from type-3 (SMB) inbound, capturing source host + account.
2. Identify the SOURCE host and account; confirm whether that account legitimately administers the target.
3. For SMB: check 5140/5145 for admin-share access and 7045/4697 for a freshly-installed service (PsExec/remote-exec).
4. For RDP: correlate 4778/1149 and look for follow-on interactive activity.
5. Build the lateral-movement graph — chain source→target hops to find the entry point and the goal host.
6. Hunt the credential's origin (likely T1003/T1078) and scope every host it logged into.

**Data sources:** Windows Security 4624 (types 3/10), 4648, 4672, 5140/5145, 4697/7045, TerminalServices Operational logs, EDR process-creation (`PSEXESVC`, `wmic`, `Invoke-Command`), internal NetFlow for 445/3389.

**Pivot/related:** T1078 (valid accounts), T1003.001 (creds used), T1569.002 (service execution / PsExec), T1570 (lateral tool transfer), T1550.002/.003 (PtH/PtT).
