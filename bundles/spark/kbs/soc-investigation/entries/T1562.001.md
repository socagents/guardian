# Impair Defenses: Disable or Modify Tools

**What it is:** Adversary disables, uninstalls, or reconfigures security tooling — AV/EDR, logging, AMSI, the firewall — to operate unseen.

**How it manifests:**
- Command lines: `Set-MpPreference -DisableRealtimeMonitoring $true`, `Add-MpPreference -ExclusionPath`, `sc stop/config <edr-service>`, `taskkill /im <edr>.exe`, `net stop`, `wevtutil cl`/`clear-eventlog` (log clearing), `fsutil usn deletejournal`, `bcdedit` tamper, `reg add ...\Windows Defender\... /v DisableAntiSpyware`.
- AMSI bypass strings in script-block logs (`amsiInitFailed`, `[Ref].Assembly.GetType('...AmsiUtils')`).
- BYOVD: a vulnerable signed driver loaded (e.g. `RTCore64.sys`, `gmer`, `Process Explorer` driver) to kill EDR from kernel.
- Telemetry gap: EDR agent heartbeat stops; Defender events 5001/5010 (real-time disabled); Security 1102 (audit log cleared).

**Investigation steps (in order):**
1. Check the EDR/AV console for the host going silent/unmanaged and the exact timestamp — that defines the blind-window start.
2. Pull process-creation just before the gap for the disabling command (`Set-MpPreference`, `sc`, `taskkill`, `wevtutil`).
3. Review Defender Operational (5001/5007/5010) + Security 1102 + System service-stop events.
4. Hunt for newly loaded unsigned/known-vulnerable drivers (Sysmon Event 6 / loaded-image) indicating BYOVD.
5. Treat the blind window as high-risk — reconstruct activity from network/auth logs that the adversary couldn't disable.
6. Identify any added AV exclusions and inspect those paths for payloads.

**Data sources:** EDR/AV management console + agent-health, Defender Operational log, Windows Security 1102, System log (service control), Sysmon image-load (Event 6/7), network logs as out-of-band corroboration.
