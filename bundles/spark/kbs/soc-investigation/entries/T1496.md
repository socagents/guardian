# Resource Hijacking (Cryptomining)

**What it is:** Adversary co-opts compromised host/cloud compute to mine cryptocurrency (or run other resource-intensive workloads), driving up cost and degrading performance.

**How it manifests:**
- Resource telemetry: sustained near-100% CPU/GPU on a host or container, often at off-hours, with no business workload to explain it; cloud bill spikes / new high-CPU instance types spun up.
- Process/command: miner binaries (`xmrig`, `minerd`, `t-rex`, `nbminer`), command lines with `--pool`, `--wallet`, a Monero/stratum address; processes masquerading as `kworker`/`systemd`/`[migration]` on Linux.
- Network: connections to mining pools on stratum ports (3333/4444/5555/14444) or `pool.*`/`xmr.*` domains; persistent outbound to mining infra.
- Persistence: cron jobs, systemd units, scheduled tasks, or run keys re-launching the miner; killing competing miners.
- Cloud: compromised IAM keys launching instances in unusual regions (T1078.004).

**Investigation steps (in order):**
1. Confirm the high-utilization process and pull its binary hash + command line (look for pool/wallet args).
2. Identify network connections to mining-pool domains/stratum ports from the host (proxy/DNS/flow).
3. Find persistence: cron/systemd/scheduled-task/run-key entries re-spawning the miner.
4. Determine entry: exploited public app (T1190), exposed Docker/K8s API, leaked cloud keys, or supply-chain image — trace upstream.
5. For cloud, review CloudTrail/audit for `RunInstances`/key usage from anomalous IPs and rotate compromised credentials.
6. Remediate the foothold, not just the miner — miners are a symptom of a broader compromise.

**Data sources:** Host/container CPU-GPU metrics, EDR process events + command lines, DNS/proxy/NetFlow (pool connections), cloud billing + CloudTrail/audit logs, cron/systemd/scheduled-task inventory.

**Pivot/related:** T1190 (initial access via exposed app), T1610 (deploy container), T1078.004 (cloud accounts), T1053/T1543 (persistence), T1496 often co-resident with broader RCE.
