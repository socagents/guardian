# OS Credential Dumping: LSASS Memory

**What it is:** Adversary reads or dumps the memory of the LSASS process to extract plaintext passwords, NTLM hashes, and Kerberos tickets.

**How it manifests:**
- Process-access events: a non-system process opening a handle to `lsass.exe` with `PROCESS_VM_READ`/`0x1010`/`0x1410` access rights (Sysmon Event 10 with TargetImage `lsass.exe`).
- Command lines / tools: `procdump -ma lsass.exe`, `rundll32 comsvcs.dll, MiniDump <pid> lsass.dmp full`, `tasklist`/`MiniDumpWriteDump`, Mimikatz (`sekurlsa::logonpasswords`), `nanodump`, `pypykatz`, or `taskmgr` right-click "Create dump file" abuse.
- File artifacts: an `lsass.dmp`/`.dmp`/randomly-named dump in `%TEMP%`, `%PUBLIC%`, or `C:\Windows\Temp`.
- Often immediately follows privilege escalation and precedes lateral movement.

**Investigation steps (in order):**
1. Pull EDR/Sysmon Event 10 for handles opened against `lsass.exe`; identify the source process and its hash/signer.
2. Check process-creation for `comsvcs.dll MiniDump`, `procdump`, Mimikatz, or `nanodump` command lines around that time.
3. Hunt the filesystem for `.dmp` files written near the event; capture and (if policy allows) inspect.
4. Confirm the actor's privilege level (LSASS read requires SYSTEM/admin) and trace how they got it.
5. Treat every credential cached on that host as compromised — enumerate logged-on users/sessions and force resets.
6. Pivot forward: look for those creds being reused (T1078, lateral RDP/SMB/WinRM).

**Data sources:** Sysmon Event 10 (ProcessAccess) + Event 11 (FileCreate for `.dmp`), EDR process-creation with command lines, Windows Security 4656/4663 (handle to LSASS object), file-creation telemetry.

**Pivot/related:** T1078 (reuse of dumped creds), T1134 (token theft), T1021 (lateral movement), T1550.002 (Pass-the-Hash), T1562.001 (often disables EDR first).
