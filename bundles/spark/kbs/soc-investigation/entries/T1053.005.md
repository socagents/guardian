# Scheduled Task/Job: Scheduled Task

**What it is:** Adversary creates or modifies a Windows Scheduled Task for persistence, privilege escalation, or recurring execution.

**How it manifests:**
- Command lines: `schtasks /create /tn <name> /tr <payload> /sc minute|onlogon|onstart /ru SYSTEM`; PowerShell `Register-ScheduledTask`. Look for tasks running scripts from `%TEMP%`/`%APPDATA%`/`%PUBLIC%`, encoded PowerShell, or LOLBins.
- Event logs: Security 4698 (task created), 4702 (updated), 4699 (deleted); Task Scheduler Operational 106/140/141/200.
- File artifacts: task XML in `C:\Windows\System32\Tasks\<name>` and registry under `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tree`.
- Naming: masquerades as legitimate (e.g. `GoogleUpdateTaskMachine`, `OneDrive Standalone Update`) but points at an unusual binary/path.

**Investigation steps (in order):**
1. Pull 4698 events; extract task name, author, principal (`/ru`), trigger, and the Action command line.
2. Evaluate the Action target — is it a signed legit updater, or a script/LOLBin in a user-writable path?
3. Cross-reference task creation time with the suspected initial-access event (same parent process chain?).
4. Inspect the task XML and registry TaskCache entry to confirm and capture the full definition.
5. Check whether the task already executed (Task Scheduler Operational 200/201) and trace child processes.
6. Hunt the same task name/payload across the fleet for campaign scope.

**Data sources:** Security event log (4698/4699/4702), Task Scheduler Operational log, EDR process-creation (`schtasks.exe`/`Register-ScheduledTask`), file-creation in `System32\Tasks`, registry-write telemetry.
