# Python Service Conventions

## Language and Tooling

- Python 3.12+ with type hints everywhere.
- Package manager: `uv` (lockfile: `uv.lock`).
- Lint/format: `ruff check . && ruff format .`
- Type check: `mypy .` (strict mode).
- Test: `pytest`

## Code Style

- All functions and methods have type annotations (parameters and return types).
- No `Any` unless interfacing with untyped third-party libraries.
- Use `snake_case` for variables, functions, modules. `PascalCase` for classes.
- Async/await patterns for I/O-bound operations.
- Use `dataclasses` or `pydantic` for structured data, not plain dicts.

## Testing

- Framework: pytest with fixtures.
- Test files are `test_*.py` or `*_test.py` colocated with source.
- Use `pytest-asyncio` for async tests.
- Mock external dependencies (LLM APIs, databases) in unit tests.
- Target 80%+ coverage on new code.

## Project Structure

- Each service has its own `pyproject.toml` under `services/<name>/`.
- Source code in `src/` within the service directory.
- Use `__init__.py` for explicit public APIs.

## LanceDB (Memory Service)

- Vector operations use LanceDB as the storage backend.
- Follow the memory-service spec for schema and query patterns.
- Embeddings are generated by the agent-runtime, not the memory service.
